# Architecture Overview

This page provides a high-level view of the clinvar-ingest-bq-tools repository structure, tech stack, and how the components fit together.

## Repository Structure

```
clinvar-ingest-bq-tools/
|-- src/                          # TypeScript source files
|   |-- bq-utils.ts               #   BigQuery utility functions (date formatting, JSON transforms)
|   |-- parse-utils.ts            #   ClinVar content parsing (AttributeSet, Citation, HGVS, etc.)
|-- dist/                         # Compiled JavaScript output (generated by build)
|-- test/                         # Jest tests and sample ClinVar XML data
|-- scripts/                      # SQL scripts organized by function (~104 files)
|   |-- clinvar-curation/         #   ClinVar curation workflow scripts
|   |-- dataset-preparation/      #   Data normalization and validation procedures
|   |-- general/                  #   Schema/release table functions and utilities
|   |-- parsing-funcs/            #   BigQuery UDF definitions for parsing
|   |-- temporal-data-collection/ #   Per-release snapshot extraction procedures
|   |-- temporal-data-summation/  #   Temporal aggregation procedures
|   |-- tracker-report-update/    #   Report generation procedures
|   |-- clinvar-miner/            #   ClinVar data mining queries
|   |-- conflict-resolution-analysis/ # Conflict resolution analytics
|   |-- mechanism-threshold/      #   Mechanism threshold analysis
|   |-- molecular-consequence/    #   Molecular consequence analysis
|   |-- projects/                 #   Project-specific queries
|   |-- gks-procs/                #   GA4GH GKS procedures
|-- gcp-services/                 # Cloud Functions
|   |-- gcs-file-ingest-service/  #   Reference data ingestion from GCS
|   |-- conflict-analytics-trigger/ # Conflict analytics trigger
|-- .github/workflows/            # CI/CD pipeline
|-- docs/                         # Documentation (MkDocs Material)
```

## Tech Stack

| Layer | Technology | Purpose |
|-------|-----------|---------|
| **UDF Logic** | TypeScript | Parsing and transformation functions compiled to JS |
| **Data Processing** | SQL (BigQuery) | Stored procedures, table functions, and analytics |
| **Cloud Functions** | Python (Flask) | Reference data ingestion from GCS into BigQuery |
| **Infrastructure** | Google Cloud Platform | BigQuery, Cloud Storage, Cloud Functions |
| **CI/CD** | GitHub Actions | Automated build, test, and deployment |
| **Testing** | Jest | Unit tests for TypeScript utilities |

## How the Pieces Fit Together

The deployment and execution flow connects the components as follows:

1. **TypeScript compiles to JavaScript** -- `src/*.ts` files are compiled via `npx tsc` to produce `dist/*.js` output files.

2. **JavaScript is uploaded to GCS** -- The CI/CD pipeline copies compiled JS files to `gs://clinvar-ingest/bq-tools`, where BigQuery can access them as UDF libraries.

3. **BigQuery UDFs reference the JS libraries** -- SQL parsing functions defined in `scripts/parsing-funcs/` create BigQuery UDFs that load the JS libraries from GCS. Functions like `parseAttributeSet`, `parseCitations`, `parseMethods`, and `parseSample` are used throughout the SQL procedures.

4. **SQL procedures call UDFs** -- Stored procedures in `scripts/` use these UDFs to parse ClinVar JSON content fields, normalize data, and build derived tables.

5. **CI/CD auto-deploys changes** -- On push to `main`, the GitHub Actions workflow:
    - Detects whether TypeScript files changed and, if so, compiles, tests, and uploads the JS to GCS
    - Detects whether SQL procedure/function files changed (`*-proc.sql`, `*-func.sql`) and executes them against BigQuery in `clingen-dev`

!!! info "Deployment Scope"
    The CI/CD pipeline deploys only to the `clingen-dev` project. Promotion to `clingen-stage` or production requires separate processes.
